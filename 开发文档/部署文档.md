# 部署文档

本文档基于当前仓库的实际可运行方式整理，适用于本项目在本机通过 Docker 打包和部署。

## 1. 当前部署约定

- 应用访问地址：`http://localhost:8080`
- 健康检查地址：`http://localhost:8080/api/health`
- MySQL 测试库端口：`3307`
- 数据库名：`bishe`
- 数据库账号：`gamestore`
- 数据库密码：`password`

说明：

- 当前 `docker-compose.yml` 中的 `app` 服务默认连接宿主机 `3307` 端口上的 MySQL。
- 因此推荐先准备好本地测试库，再启动应用容器。

## 2. 前置条件

在项目根目录执行以下命令前，请确保：

- 已安装 Docker Desktop 或等价 Docker 环境
- Docker 能正常启动
- 当前目录为项目根目录

进入项目根目录：

```bash
cd /Users/shuo/Documents/projects/bishe
```

## 3. 初始化测试数据库

如果本机还没有用于演示的测试库，可直接使用 `开发文档/sql` 里的完整备份初始化一个 MySQL 容器。

启动数据库容器：

```bash
docker run -d \
  --name bishe-mysql-3307 \
  -p 3307:3306 \
  -e TZ=Asia/Shanghai \
  -e MYSQL_DATABASE=bishe \
  -e MYSQL_USER=gamestore \
  -e MYSQL_PASSWORD=password \
  -e MYSQL_ROOT_PASSWORD=rootpassword \
  -v bishe_mysql_data_3307:/var/lib/mysql \
  -v /Users/shuo/Documents/projects/bishe/开发文档/sql/bishe_full_notablespaces_20260416_083830.sql:/docker-entrypoint-initdb.d/01_bishe_full.sql:ro \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci
```

查看数据库容器状态：

```bash
docker ps
```

查看数据库初始化日志：

```bash
docker logs -f bishe-mysql-3307
```

如果只想导入数据、不重建表结构，可参考：

- [README.txt](/Users/shuo/Documents/projects/bishe/开发文档/sql/README.txt)
- [bishe_full_notablespaces_20260416_083830.sql](/Users/shuo/Documents/projects/bishe/开发文档/sql/bishe_full_notablespaces_20260416_083830.sql)
- [bishe_data_only_notablespaces_20260416_083830.sql](/Users/shuo/Documents/projects/bishe/开发文档/sql/bishe_data_only_notablespaces_20260416_083830.sql)

## 4. 打包 Docker 镜像

本项目的 `Dockerfile` 会在镜像构建过程中自动执行 Maven 打包，因此不需要先手工在宿主机执行 `mvn package`。

单独打包镜像：

```bash
docker build -t game-community-store:local .
```

当前 `Dockerfile` 路径：

- [Dockerfile](/Users/shuo/Documents/projects/bishe/Dockerfile)

## 5. 部署应用

### 5.1 首次部署或普通启动

推荐命令：

```bash
docker compose up -d --build app
```

该命令会：

- 读取当前代码
- 构建最新应用镜像
- 启动 `app` 容器

### 5.2 项目更新后重新部署

如果代码已经更新，建议强制重建容器：

```bash
docker compose up -d --build --force-recreate app
```

当前编排文件路径：

- [docker-compose.yml](/Users/shuo/Documents/projects/bishe/docker-compose.yml)

## 8. 常用运维命令

停止应用容器：

```bash
docker compose stop app
```

启动已存在的应用容器：

```bash
docker compose start app
```

停止并移除应用容器与网络：

```bash
docker compose down
```

删除应用镜像后重新完整构建：

```bash
docker compose down
docker rmi game-community-store:local
docker compose up -d --build app
```

停止并删除测试数据库容器：

```bash
docker rm -f bishe-mysql-3307
```

如果连数据库卷也一并删除：

```bash
docker rm -f bishe-mysql-3307
docker volume rm bishe_mysql_data_3307
```

## 10. 补充说明

- 当前应用默认连接 `localhost:3307`，对应配置见 [application.properties](/Users/shuo/Documents/projects/bishe/src/main/resources/application.properties)。
- 若你改了数据库地址、账号或密码，可在启动前通过环境变量覆盖：

```bash
export SPRING_DATASOURCE_URL='jdbc:mysql://localhost:3307/bishe?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true'
export SPRING_DATASOURCE_USERNAME='gamestore'
export SPRING_DATASOURCE_PASSWORD='password'
docker compose up -d --build --force-recreate app
```

- 当前 `docker-compose.yml` 中的 `mysql` 服务带有 `with-local-db` profile，默认不会自动启动；当前主流程仍以宿主机 `3307` 测试库为准。
