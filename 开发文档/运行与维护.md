# 运行与维护

## 1. 环境要求

- Java 21+
- Maven 3.6+
- MySQL 8.x
- 推荐使用 IntelliJ IDEA 或等价 IDE

## 2. 首次初始化

### 步骤 1：创建表结构

```bash
mysql -u root -p < 开发文档/SQL脚本/01_数据库初始化.sql
```

### 步骤 2：导入基础演示数据

```bash
mysql -u root -p bishe < 开发文档/SQL脚本/02_基础演示数据.sql
```

### 步骤 3：如果是旧环境升级，再执行修复脚本

```bash
mysql -u root -p bishe < 开发文档/SQL脚本/03_结构修复与数据整理.sql
```

## 3. 启动项目

### Maven 方式

```bash
mvn spring-boot:run
```

### IDE 方式

- 主类：`src/main/java/com/gamestore/GameCommunityStoreApplication.java`
- 如果 IDE 仍指向旧主类，请手动改回 `com.gamestore.GameCommunityStoreApplication`

## 4. 常用访问入口

- 首页：`http://localhost:8080/`
- 游戏详情示例：`http://localhost:8080/game/1`
- 登录页：`http://localhost:8080/login`
- 注册页：`http://localhost:8080/register`
- 个人中心：`http://localhost:8080/profile`
- 社区首页：`http://localhost:8080/community`
- 讨论广场：`http://localhost:8080/forum.html`
- 后台首页：`http://localhost:8080/admin`
- 健康检查：`http://localhost:8080/api/health`

## 5. 基础验收清单

- 首页能正常加载轮播图、分类和游戏列表
- 游戏详情页能读取数据库内容
- 注册、登录、退出可正常执行
- 社区首页能显示板块列表
- 讨论广场能加载帖子与评论
- 后台页面能获取游戏、用户、社区数据

## 6. 常见问题

### 6.1 SQL 导入报 JSON / TEXT 类型错误

原因：

- 旧环境里 `games.system_requirements`、`games.tags` 可能不是当前文本结构

处理：

- 执行 `SQL脚本/03_结构修复与数据整理.sql`

### 6.2 登录失败，管理员账号不存在

原因：

- 没有导入基础数据，或旧环境被覆盖

处理：

- 重新执行 `SQL脚本/02_基础演示数据.sql`
- 默认演示账号见该脚本注释

### 6.3 分类显示重复或游戏分类不生效

原因：

- 旧环境中存在重复分类，或 `game_categories` 没有从 `games.category_id` 回填

处理：

- 先执行 `SQL脚本/03_结构修复与数据整理.sql`
- 再人工检查 `categories` 与 `game_categories`

### 6.4 IDE 启动时仍引用旧主类

处理：

- 删除旧 Run Configuration
- 重新指定 `GameCommunityStoreApplication`

## 7. 当前维护建议

### 配置层

- 开发期可保留 `spring.jpa.hibernate.ddl-auto=update`
- 准备交付时建议改为 `validate`，并完全依赖 SQL 脚本控表

### 安全层

- 当前密码为明文存储与比对，仅适合本地演示
- 建议后续接入密码加密、权限校验、统一认证中间层

### 代码层

- 调试控制器建议从主线移除或加开关
- Redis、JWT、Security、Validation 相关配置若暂不启用，建议清理为单一方案

### 数据层

- 不再继续维护根目录历史 SQL
- 新增数据修复动作时，优先补充到本目录脚本或单独新建迁移脚本
