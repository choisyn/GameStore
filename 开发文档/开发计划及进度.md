# 开发计划及进度

本文档用于替代根目录原有的“项目状态总结 / 完成确认 / 功能总结 / 修复说明 / 启动说明”等重复记录，只保留当前仍有参考价值的主线信息。

## 1. 项目定位

- 项目名称：游戏社区商城系统（Game Community Store）
- 技术路线：Spring Boot 3.2.0 + Spring Data JPA + Thymeleaf + MySQL + Bootstrap 5
- 当前形态：单体 Web 项目，前台商城、用户登录、社区、后台管理共用一套后端服务
- 当前状态：项目主体已可运行，首页、详情、登录注册、购物车、订单、游戏库、积分、社区、后台等主线能力已打通，支付网关与自动化测试仍有继续完善空间

## 4. 当前优先级建议

1. 固化数据库策略：当前 `spring.jpa.hibernate.ddl-auto=update` 适合开发，不适合长期交付。
2. 继续强化安全：已完成密码加密，但 Cookie 策略、后台权限校验和审计日志仍可增强。
3. 收尾支付表达：当前采用模拟支付流程，若做展示可继续补充更完整的支付状态流转。
4. 清理调试痕迹：`DebugController`、`TestController`、`SimpleTestController`、`TestApiController` 建议在交付前下线或隔离。
5. 补齐自动化测试与部署脚本：当前以功能可用为主，回归验证与交付脚本仍需加强。

## 5. 当前已知技术债

- 认证流程已可用，密码已改为加密存储，但权限边界仍需继续收紧。
- 数据库结构经历过多轮修补，历史脚本很多，必须以本目录的新脚本为准。
- 游戏分类同时保留了 `games.category_id` 与 `game_categories` 两套关系，现阶段属于兼容方案。
- 轮播图、社区、讨论广场存在历史脚本与现代码并行的情况，后续应继续删减旧分支。
- 模拟支付、积分规则和商城运营规则目前采用毕业设计演示方案，后续若商用仍需重构。
- 旧文档里出现过 Java 23、JSON 字段、老主类等说法，当前均不应再作为最终依据。
