# 项目框架

## 1. 技术栈概览

- 后端：Spring Boot 3.2.0、Spring Web、Spring Data JPA
- 模板层：Thymeleaf
- 前端：Bootstrap 5、Bootstrap Icons、原生 JavaScript
- 数据库：MySQL 8.x
- 构建工具：Maven
- Java 版本：`pom.xml` 当前配置为 Java 21

## 2. 当前运行特征

- 项目是单体应用，不区分独立前后端部署
- 页面路由与 REST API 同时存在于同一服务
- `application.properties` 当前使用 MySQL + JPA，`ddl-auto=update`
- Spring Security、Validation、Redis、JWT 相关内容仍有保留痕迹，但并非完整启用状态
- 商城闭环现已补充购物车、订单、游戏库和积分模块

## 3. 分层结构

### Controller 层

- 负责页面路由、接口入参与响应输出
- 主要分为前台、认证、社区、后台、调试五组

### Service 层

- 负责业务逻辑、校验、数据组装
- 主要包含游戏、分类、认证、用户、帖子、社区、轮播图服务

### Repository 层

- 负责数据库访问
- 以 Spring Data JPA 接口为主

### Entity 层

- 负责数据库映射
- 当前核心实体包括用户、游戏、分类、帖子、评论、轮播图、社区板块、社区帖子、社区评论、用户会话

### DTO / Exception / Util

- DTO 负责请求与响应模型
- Exception 负责统一异常处理
- Util 负责统一响应工具

## 5. 模块划分

### 5.1 商城前台

- 页面：`/`、`/game/{id}`
- 接口：`/api/games`、`/api/categories`、`/api/banners`
- 目标：游戏展示、详情查看、分类筛选、搜索、轮播推荐

### 5.2 用户与认证

- 页面：`/login`、`/register`、`/profile`
- 接口：`/api/auth/*`
- 目标：注册、登录、退出、当前用户查询、会话管理、密码加密、个人资料修改、密码修改

### 5.3 商城闭环

- 页面：`/cart`、`/orders`、`/library`
- 接口：`/api/store/*`
- 目标：加入购物车、模拟支付、订单查询、自动入库、积分累计与用户资产展示

### 5.4 讨论广场

- 页面：`/forum.html`、`/post-detail.html`、`/create-post.html`
- 接口：`/api/posts/*`
- 目标：发帖、评论、搜索、帖子详情、用户互动

### 5.5 社区板块

- 页面：`/community`
- 接口：`/api/community/*`
- 目标：展示板块、板块帖子、社区评论、社区内容管理

### 5.6 后台管理

- 页面：`/admin`
- 接口：
  - `/api/admin/dashboard/*`
  - `/api/admin/games/*`
  - `/api/admin/users/*`
  - `/api/admin/community/*`
- 目标：游戏管理、用户管理、社区管理、仪表盘概览统计

### 5.7 调试与临时接口

- `DebugController`
- `SimpleTestController`
- `TestController`
- `TestApiController`

这部分主要服务于开发联调，不建议在最终交付版继续暴露。

## 6. 关键设计说明

### 游戏分类

- 当前同时保留 `games.category_id` 与 `game_categories`
- `category_id` 负责兼容旧代码和部分查询
- `game_categories` 负责多分类扩展

### 登录态

- 当前使用 `user_sessions` 持久化 Session Token
- 前端多页面共享登录状态，但历史上出现过 Cookie 读取、菜单渲染不一致问题

### 前台页面形态

- 一部分页面走 Thymeleaf 模板
- 一部分社区页面仍以 `static/*.html` 方式直出
- 这是当前结构中的一个混合点，后续可再统一

## 7. 建议后续重构方向

- 将社区相关页面路由统一到模板体系或前后端分离体系，不再混用
- 将后台管理 API 与前台 API 进一步按模块隔离
- 将测试、调试控制器迁出主线代码
- 将安全配置与会话策略正式化，减少“预留配置但未完全接通”的状态
