# 后台管理系统说明文档

## 概述
游戏社区商城后台管理系统，用于管理游戏、社区、用户等核心功能。

## 访问地址
- URL: `http://localhost:8080/admin`
- 登录账号: `admin`
- 登录密码: `admin`

## 功能模块

### 1. 仪表盘 (Dashboard)
显示系统核心数据统计：
- 总用户数 / 活跃用户数
- 游戏总数
- 社区帖子统计
- 销售额统计

**实现文件**:
- 前端: `src/main/resources/static/js/admin/dashboard.js`
- 后端API: 各模块的stats接口

### 2. 游戏管理 (Game Management)
管理平台上的所有游戏，支持完整的CRUD操作。

**功能列表**:
- ✅ 查看所有游戏列表（分页显示）
- ✅ 添加新游戏
- ✅ 编辑游戏信息
  - 游戏名称、分类
  - 描述、封面图
  - 价格、折扣价
  - 评分、发行日期
  - 系统要求、标签
  - 上架/下架状态
  - 精选游戏设置
- ✅ 删除游戏（软删除，设为INACTIVE）
- ✅ 批量删除

**实现文件**:
- 前端: `src/main/resources/static/js/admin/games.js`
- 后端API: `src/main/java/com/gamestore/controller/AdminGameController.java`
- 服务层: `src/main/java/com/gamestore/service/GameService.java`

**API接口**:
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/admin/games` | 获取所有游戏（分页） |
| GET | `/api/admin/games/{id}` | 获取游戏详情 |
| POST | `/api/admin/games` | 创建新游戏 |
| PUT | `/api/admin/games/{id}` | 更新游戏信息 |
| DELETE | `/api/admin/games/{id}` | 删除游戏 |
| DELETE | `/api/admin/games/batch` | 批量删除游戏 |

### 3. 社区管理 (Community Management)
管理社区帖子和评论，包括审核、置顶、精华等功能。

**功能列表**:

#### 帖子管理
- ✅ 查看所有帖子列表
- ✅ 查看帖子详情
- ✅ 设置/取消精华帖
- ✅ 设置/取消置顶帖
- ✅ 关闭/开启帖子评论
- ✅ 隐藏帖子
- ✅ 删除帖子（软删除）

#### 评论管理
- ✅ 查看所有评论
- ✅ 删除违规评论

**实现文件**:
- 前端: `src/main/resources/static/js/admin/community.js`
- 后端API: `src/main/java/com/gamestore/controller/AdminCommunityController.java`
- 服务层: `src/main/java/com/gamestore/service/CommunityService.java`

**API接口**:
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/admin/community/posts` | 获取所有帖子 |
| GET | `/api/admin/community/posts/{id}` | 获取帖子详情 |
| PUT | `/api/admin/community/posts/{id}/essence` | 设置精华 |
| PUT | `/api/admin/community/posts/{id}/pin` | 置顶帖子 |
| PUT | `/api/admin/community/posts/{id}/close` | 关闭评论 |
| PUT | `/api/admin/community/posts/{id}/hide` | 隐藏帖子 |
| DELETE | `/api/admin/community/posts/{id}` | 删除帖子 |
| GET | `/api/admin/community/comments` | 获取所有评论 |
| DELETE | `/api/admin/community/comments/{id}` | 删除评论 |

### 4. 用户管理 (User Management)
管理平台用户，包括权限控制和账号状态管理。

**功能列表**:
- ✅ 查看所有用户列表
- ✅ 查看用户详情
- ✅ 编辑用户信息
  - 邮箱
  - 角色（普通用户/管理员）
  - 状态（正常/封禁/删除）
- ✅ 修改用户角色
- ✅ 封禁/解封用户
- ✅ 删除用户
- ✅ 重置用户密码
- ✅ 查看用户统计信息

**实现文件**:
- 前端: `src/main/resources/static/js/admin/users.js`
- 后端API: `src/main/java/com/gamestore/controller/AdminUserController.java`
- 服务层: `src/main/java/com/gamestore/service/UserService.java`

**API接口**:
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/admin/users` | 获取所有用户 |
| GET | `/api/admin/users/{id}` | 获取用户详情 |
| PUT | `/api/admin/users/{id}` | 更新用户信息 |
| PUT | `/api/admin/users/{id}/role` | 修改用户角色 |
| PUT | `/api/admin/users/{id}/status` | 修改用户状态 |
| DELETE | `/api/admin/users/{id}` | 删除用户 |

## 权限控制

### 访问权限
- 后台管理系统只允许**管理员角色（ADMIN）**访问
- 登录验证通过 `AuthController.getCurrentUser()` 实现
- 未登录或非管理员用户将被重定向到登录页

### 角色类型
- `USER`: 普通用户
- `ADMIN`: 管理员
- `MODERATOR`: 版主（预留）

## 技术实现

### 前端技术
- **Bootstrap 5**: UI框架
- **Bootstrap Icons**: 图标库
- **原生JavaScript**: 页面交互逻辑
- **模态框**: 使用Bootstrap Modal实现编辑功能

### 后端技术
- **Spring Boot**: 框架
- **Spring Data JPA**: 数据持久化
- **RESTful API**: 接口设计
- **Session Token**: 会话管理

### 数据库
- **MySQL**: 主数据库
- 相关表：
  - `games`: 游戏表
  - `users`: 用户表
  - `user_sessions`: 会话表
  - `community_posts`: 帖子表
  - `community_comments`: 评论表
  - `community_sections`: 板块表

## 使用说明

### 1. 登录后台
1. 访问 `http://localhost:8080/admin`
2. 如未登录，会自动跳转到登录页
3. 使用管理员账号登录：`admin` / `admin`

### 2. 游戏管理
1. 点击左侧菜单"游戏管理"
2. 查看游戏列表
3. 点击"添加游戏"创建新游戏
4. 点击"编辑"按钮修改游戏信息
5. 点击"删除"按钮下架游戏

### 3. 社区管理
1. 点击左侧菜单"社区管理"
2. 切换"帖子管理"或"评论管理"标签
3. 对违规内容进行删除
4. 设置优质帖子为精华或置顶

### 4. 用户管理
1. 点击左侧菜单"用户管理"
2. 查看所有用户列表
3. 编辑用户角色和状态
4. 封禁违规用户
5. 重置用户密码

## 注意事项

1. **权限验证**: 所有后台操作都需要管理员权限
2. **软删除**: 删除操作通常是软删除，不会真正从数据库删除数据
3. **数据一致性**: 删除帖子时，相关的评论、点赞等数据也应该处理
4. **安全性**: 
   - 密码重置功能应该加密存储
   - 建议添加操作日志
   - 建议添加二次确认
