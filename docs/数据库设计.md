# 游戏社区商城系统 - 数据库设计文档

## 数据库概述
- **数据库类型**: MySQL 8.0+
- **数据库名称**: bishe
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **存储引擎**: InnoDB

## ⚠️ 重要提示

### 数据完整性约束

1. **游戏名称唯一性**
   - 系统在创建和更新游戏时会自动检查名称是否重复
   - 如果名称已存在,将返回错误提示
   - 维护脚本: `删除重复游戏.sql`

2. **标签格式规范**
   - 游戏标签必须使用逗号分隔格式: `标签1,标签2,标签3`
   - 不要使用JSON数组格式: `["标签1", "标签2"]`
   - 如果导入数据后标签格式不对,执行 `修复游戏标签格式.sql`

## 数据库创建

### 创建数据库
```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS bishe 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE bishe;

-- 创建用户（可选）
CREATE USER IF NOT EXISTS 'gamestore'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON bishe.* TO 'gamestore'@'localhost';
FLUSH PRIVILEGES;
```

## 数据表设计

### 1. 用户表 (users)
```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL COMMENT '用户名',
    email VARCHAR(100) UNIQUE NOT NULL COMMENT '邮箱',
    password VARCHAR(255) NOT NULL COMMENT '密码(加密)',
    avatar VARCHAR(255) COMMENT '头像URL',
    role ENUM('USER', 'ADMIN', 'MODERATOR') DEFAULT 'USER' COMMENT '用户角色',
    status ENUM('ACTIVE', 'INACTIVE', 'BANNED') DEFAULT 'ACTIVE' COMMENT '用户状态',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    last_login_at TIMESTAMP NULL COMMENT '最后登录时间',
    INDEX idx_username (username),
    INDEX idx_email (email),
    INDEX idx_status (status)
) COMMENT '用户表';
```

### 2. 分类表 (categories)
```sql
CREATE TABLE categories (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '分类名称',
    description TEXT COMMENT '分类描述',
    parent_id BIGINT NULL COMMENT '父分类ID',
    sort_order INT DEFAULT 0 COMMENT '排序',
    status ENUM('ACTIVE', 'INACTIVE') DEFAULT 'ACTIVE' COMMENT '状态',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES categories(id) ON DELETE SET NULL,
    INDEX idx_parent_id (parent_id),
    INDEX idx_sort_order (sort_order),
    INDEX idx_status (status)
) COMMENT '分类表';
```

**字段说明**:
- `status`: 使用ENUM类型('ACTIVE', 'INACTIVE')替代布尔值,更符合实体设计

### 3. 游戏表 (games)
```sql
CREATE TABLE games (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(200) NOT NULL COMMENT '游戏名称',
    description TEXT COMMENT '游戏描述',
    category_id BIGINT NOT NULL COMMENT '分类ID',
    developer VARCHAR(100) COMMENT '开发商',
    publisher VARCHAR(100) COMMENT '发行商',
    release_date DATE COMMENT '发布日期',
    price DECIMAL(10,2) DEFAULT 0.00 COMMENT '价格',
    discount_price DECIMAL(10,2) NULL COMMENT '折扣价格',
    image_url VARCHAR(255) COMMENT '主图片URL',
    gallery JSON COMMENT '图片集合',
    system_requirements JSON COMMENT '系统要求',
    tags JSON COMMENT '标签',
    rating DECIMAL(3,2) DEFAULT 0.00 COMMENT '评分',
    rating_count INT DEFAULT 0 COMMENT '评分人数',
    download_count INT DEFAULT 0 COMMENT '下载次数',
    is_featured BOOLEAN DEFAULT FALSE COMMENT '是否推荐',
    status ENUM('ACTIVE', 'INACTIVE', 'COMING_SOON') DEFAULT 'ACTIVE',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES categories(id),
    INDEX idx_category_id (category_id),
    INDEX idx_price (price),
    INDEX idx_rating (rating),
    INDEX idx_status (status),
    FULLTEXT idx_name_desc (name, description)
) COMMENT '游戏表';
```

**字段说明**:
- `category_id`: 保留用于向后兼容,实际分类关系使用`game_categories`表
- 支持多对多分类关系

### 3.1. 游戏-分类关联表 (game_categories)
```sql
CREATE TABLE game_categories (
    game_id BIGINT NOT NULL COMMENT '游戏ID',
    category_id BIGINT NOT NULL COMMENT '分类ID',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (game_id, category_id),
    FOREIGN KEY (game_id) REFERENCES games(id) ON DELETE CASCADE,
    FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE CASCADE,
    INDEX idx_category_id (category_id),
    INDEX idx_game_id (game_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='游戏与分类关联表(多对多)';
```

**设计说明**:
- 实现游戏与分类的多对多关系
- 一个游戏可以属于多个分类
- 联合主键确保关系唯一性
- 级联删除保证数据一致性

### 4. 商品表 (products)
```sql
CREATE TABLE products (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(200) NOT NULL COMMENT '商品名称',
    description TEXT COMMENT '商品描述',
    category_id BIGINT NOT NULL COMMENT '分类ID',
    game_id BIGINT NULL COMMENT '关联游戏ID',
    price DECIMAL(10,2) NOT NULL COMMENT '价格',
    discount_price DECIMAL(10,2) NULL COMMENT '折扣价格',
    stock_quantity INT DEFAULT 0 COMMENT '库存数量',
    image_url VARCHAR(255) COMMENT '主图片URL',
    gallery JSON COMMENT '图片集合',
    attributes JSON COMMENT '商品属性',
    is_digital BOOLEAN DEFAULT TRUE COMMENT '是否数字商品',
    is_featured BOOLEAN DEFAULT FALSE COMMENT '是否推荐',
    status ENUM('ACTIVE', 'INACTIVE', 'OUT_OF_STOCK') DEFAULT 'ACTIVE',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES categories(id),
    FOREIGN KEY (game_id) REFERENCES games(id) ON DELETE SET NULL,
    INDEX idx_category_id (category_id),
    INDEX idx_game_id (game_id),
    INDEX idx_price (price),
    INDEX idx_status (status),
    FULLTEXT idx_name_desc (name, description)
) COMMENT '商品表';
```

### 5. 订单表 (orders)
```sql
CREATE TABLE orders (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_number VARCHAR(50) UNIQUE NOT NULL COMMENT '订单号',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    total_amount DECIMAL(10,2) NOT NULL COMMENT '订单总金额',
    discount_amount DECIMAL(10,2) DEFAULT 0.00 COMMENT '折扣金额',
    final_amount DECIMAL(10,2) NOT NULL COMMENT '实付金额',
    payment_method ENUM('CREDIT_CARD', 'PAYPAL', 'ALIPAY', 'WECHAT') COMMENT '支付方式',
    payment_status ENUM('PENDING', 'PAID', 'FAILED', 'REFUNDED') DEFAULT 'PENDING',
    order_status ENUM('PENDING', 'CONFIRMED', 'PROCESSING', 'SHIPPED', 'DELIVERED', 'CANCELLED') DEFAULT 'PENDING',
    shipping_address JSON COMMENT '收货地址',
    notes TEXT COMMENT '订单备注',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    paid_at TIMESTAMP NULL COMMENT '支付时间',
    shipped_at TIMESTAMP NULL COMMENT '发货时间',
    delivered_at TIMESTAMP NULL COMMENT '收货时间',
    FOREIGN KEY (user_id) REFERENCES users(id),
    INDEX idx_user_id (user_id),
    INDEX idx_order_number (order_number),
    INDEX idx_payment_status (payment_status),
    INDEX idx_order_status (order_status),
    INDEX idx_created_at (created_at)
) COMMENT '订单表';
```

### 6. 订单项表 (order_items)
```sql
CREATE TABLE order_items (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_id BIGINT NOT NULL COMMENT '订单ID',
    product_id BIGINT NOT NULL COMMENT '商品ID',
    product_name VARCHAR(200) NOT NULL COMMENT '商品名称(快照)',
    product_image VARCHAR(255) COMMENT '商品图片(快照)',
    unit_price DECIMAL(10,2) NOT NULL COMMENT '单价',
    quantity INT NOT NULL COMMENT '数量',
    total_price DECIMAL(10,2) NOT NULL COMMENT '小计',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES orders(id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES products(id),
    INDEX idx_order_id (order_id),
    INDEX idx_product_id (product_id)
) COMMENT '订单项表';
```

### 7. 购物车表 (cart_items)
```sql
CREATE TABLE cart_items (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    product_id BIGINT NOT NULL COMMENT '商品ID',
    quantity INT NOT NULL DEFAULT 1 COMMENT '数量',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE,
    UNIQUE KEY uk_user_product (user_id, product_id),
    INDEX idx_user_id (user_id)
) COMMENT '购物车表';
```

### 8. 帖子表 (posts)
```sql
CREATE TABLE posts (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
```

## 索引策略

### 主要索引
1. **主键索引**: 所有表的 `id` 字段
2. **唯一索引**: `username`, `email`, `order_number` 等
3. **外键索引**: 所有外键字段
4. **复合索引**: `(user_id, product_id)` 等
5. **全文索引**: 商品名称、描述等搜索字段

### 性能优化索引
- 按创建时间查询: `idx_created_at`
- 按状态查询: `idx_status`
- 按价格范围查询: `idx_price`
- 按分类查询: `idx_category_id`

### 备份脚本
```bash
#!/bin/bash
# 数据库备份脚本
DATE=$(date +%Y%m%d_%H%M%S)
mysqldump -u gamestore -p bishe > backup_$DATE.sql
```

---

**注意**: 
1. 所有时间字段使用 `TIMESTAMP` 类型，默认为当前时间
2. 金额字段使用 `DECIMAL(10,2)` 类型，避免浮点数精度问题
3. 状态字段使用 `ENUM` 类型，提高查询效率
4. JSON 字段用于存储灵活的结构化数据
5. 外键约束确保数据完整性
